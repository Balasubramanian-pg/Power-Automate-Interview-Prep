# 121. Best Practices: Clear Markdown Formatting

Canonical documentation for 121. Best Practices: Clear Markdown Formatting. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 121. Best Practices: Clear Markdown Formatting exists and the class of problems it addresses.
The purpose of 121. Best Practices: Clear Markdown Formatting is to provide a set of guidelines for creating readable, consistent, and well-structured Markdown documents. This topic addresses the class of problems related to poorly formatted Markdown documents, including difficulty in reading and understanding the content, inconsistencies in formatting, and lack of standardization.

## 2. Conceptual Overview
Provide a high-level mental model of the topic.
Clear Markdown formatting is based on the idea of using a simple, consistent, and readable syntax to create documents that are easy to understand and maintain. This involves using headings, paragraphs, lists, and other formatting elements in a way that is consistent with standard Markdown conventions.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| Markdown | A lightweight markup language used for creating formatted text |
| Heading | A title or subtitle in a document, denoted by one or more `#` characters |
| Paragraph | A block of text separated from other blocks by a blank line |
| List | A collection of items, denoted by `*`, `-`, or `+` characters |
| Code Block | A block of text denoted by triple backticks or indentation, used for displaying code |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic.
The core concepts of Clear Markdown Formatting include:
* Consistency: Using consistent formatting throughout a document to create a clear and readable structure.
* Readability: Using headings, paragraphs, and lists to create a document that is easy to read and understand.
* Simplicity: Avoiding unnecessary complexity and using simple, intuitive syntax to create formatted text.

## 5. Standard Model
Describe the generally accepted or recommended model.
The standard model for Clear Markdown Formatting involves using:
* Headings to create a hierarchical structure for the document
* Paragraphs to separate blocks of text and create a clear flow of ideas
* Lists to present collections of items in a clear and concise way
* Code blocks to display code examples and other formatted text

## 6. Common Patterns
Document recurring, accepted patterns.
Common patterns in Clear Markdown Formatting include:
* Using `#` characters to denote headings
* Using `*`, `-`, or `+` characters to denote list items
* Using triple backticks or indentation to denote code blocks
* Using blank lines to separate paragraphs and create a clear structure

## 7. Anti-Patterns
Describe common but discouraged practices.
Anti-patterns in Clear Markdown Formatting include:
* Using inconsistent formatting throughout a document
* Using excessive or unnecessary markup to create complex formatting
* Failing to use headings, paragraphs, and lists to create a clear structure
* Using ambiguous or unclear syntax to create formatted text
