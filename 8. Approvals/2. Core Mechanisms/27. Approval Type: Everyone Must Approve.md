# 27. Approval Type: Everyone Must Approve

Canonical documentation for 27. Approval Type: Everyone Must Approve. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 27. Approval Type: Everyone Must Approve exists and the class of problems it addresses.
The "Everyone Must Approve" approval type is designed to ensure that all designated approvers consent to a particular action, decision, or request. This approval type addresses scenarios where collective agreement is mandatory, such as financial transactions, critical business decisions, or sensitive information sharing. By requiring unanimous approval, organizations can mitigate risks, ensure compliance, and maintain transparency.

## 2. Conceptual Overview
Provide a high-level mental model of the topic.
The "Everyone Must Approve" approval type involves a sequential or parallel review process where all assigned approvers must provide their consent before the request can proceed. This model typically includes the following components: request submission, approver assignment, approval notifications, and a tracking mechanism to monitor the approval status.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| Approver | An individual or entity responsible for reviewing and approving a request. |
| Approval Threshold | The minimum number of approvers required to approve a request, which is set to 100% for the "Everyone Must Approve" type. |
| Request | A submission or proposal that requires approval, such as a financial transaction or business decision. |
| Approval Workflow | A sequence of steps or tasks that a request must go through to obtain approval. |
| Unanimous Approval | A requirement that all assigned approvers must provide their consent for a request to be approved. |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic.
The core concepts of the "Everyone Must Approve" approval type include:
* Unanimous approval requirement: All assigned approvers must provide their consent.
* Approver assignment: Designated individuals or entities are responsible for reviewing and approving requests.
* Approval tracking: A mechanism to monitor the approval status and notify approvers and requesters.
* Sequential or parallel review: Approvers can review requests in a sequential or parallel manner, depending on the workflow configuration.

## 5. Standard Model
Describe the generally accepted or recommended model.
The standard model for the "Everyone Must Approve" approval type involves the following steps:
1. Request submission: A requester submits a proposal or request.
2. Approver assignment: The system assigns the request to designated approvers.
3. Approval notifications: Approvers receive notifications to review the request.
4. Approval tracking: The system monitors the approval status and updates the requesters and approvers.
5. Unanimous approval: All approvers must provide their consent for the request to be approved.

## 6. Common Patterns
Document recurring, accepted patterns.
Common patterns for the "Everyone Must Approve" approval type include:
* Using this approval type for high-risk or high-impact decisions, such as financial transactions or critical business decisions.
* Implementing a sequential review process for complex requests that require multiple levels of approval.
* Configuring parallel review for requests that require simultaneous approval from multiple approvers.

## 7. Anti-Patterns
Describe common but discouraged practices.
Anti-patterns for the "Everyone Must Approve" approval type include:
* Using this approval type for low-risk or routine decisions, which can lead to unnecessary delays and inefficiencies.
* Assigning too many approvers, which can result in approval bottlenecks and decreased productivity.
* Failing to provide clear approval criteria or guidelines, which can lead to confusion and inconsistent approval decisions.
