# 47. Item Link Description Best Practices

Canonical documentation for 47. Item Link Description Best Practices. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 47. Item Link Description Best Practices exists and the class of problems it addresses.
The purpose of Item Link Description Best Practices is to provide a standardized approach to creating descriptive links between items, enhancing user experience, and improving discoverability. This addresses the problem of unclear or misleading links, which can lead to user frustration, decreased engagement, and increased bounce rates.

## 2. Conceptual Overview
Provide a high-level mental model of the topic.
Item Link Description Best Practices revolves around the idea of creating descriptive, concise, and consistent links that accurately convey the content or action associated with an item. This involves considering the context, audience, and purpose of the link to ensure that it is intuitive, accessible, and usable.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| Item | A discrete object or entity that can be linked to, such as a webpage, document, or application. |
| Link | A clickable element that connects an item to another item or resource. |
| Description | A brief text that provides context and explains the purpose or content of a linked item. |
| Accessibility | The practice of making links and descriptions usable by everyone, including people with disabilities. |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic.
The core concepts of Item Link Description Best Practices include:
* Clarity: Links and descriptions should be easy to understand and free of ambiguity.
* Consistency: Links and descriptions should follow a consistent format and style throughout an application or website.
* Accessibility: Links and descriptions should be usable by everyone, including people with disabilities.
* Contextuality: Links and descriptions should be relevant to the surrounding content and user's goals.

## 5. Standard Model
Describe the generally accepted or recommended model.
The standard model for Item Link Description Best Practices involves:
1. Using descriptive text for links, rather than generic text like "Click here".
2. Providing a clear and concise description of the linked item.
3. Using a consistent format and style for links and descriptions.
4. Ensuring that links and descriptions are accessible to everyone, including people with disabilities.

## 6. Common Patterns
Document recurring, accepted patterns.
Common patterns for Item Link Description Best Practices include:
* Using action-oriented verbs like "Learn more" or "Download now".
* Providing a brief summary or teaser of the linked content.
* Using icons or images to supplement link text and descriptions.
* Using A/B testing to optimize link text and descriptions.

## 7. Anti-Patterns
Describe common but discouraged practices.
Anti-patterns for Item Link Description Best Practices include:
* Using generic text like "Click here" or "More information".
* Providing unclear or misleading descriptions of linked items.
* Using links that are not descriptive or do not provide context.
* Ignoring accessibility guidelines for links and descriptions.
