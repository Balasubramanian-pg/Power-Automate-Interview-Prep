# 104. Approval Type Everyone Must Approve

Canonical documentation for 104. Approval Type Everyone Must Approve. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 104. Approval Type Everyone Must Approve exists and the class of problems it addresses.
The Approval Type Everyone Must Approve exists to ensure that all stakeholders or team members approve a particular request, document, or process before it can proceed. This approval type addresses the problem of ensuring consensus and buy-in from all relevant parties, thereby reducing the risk of errors, miscommunications, or unauthorized actions.

## 2. Conceptual Overview
Provide a high-level mental model of the topic.
The Approval Type Everyone Must Approve is a type of approval workflow that requires unanimous consent from all designated approvers. This workflow is typically used in situations where multiple stakeholders have a vested interest in the outcome or where the decision has significant implications.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| Approval Type | A category of approval workflows that define the rules and requirements for obtaining approval |
| Everyone Must Approve | A specific approval type that requires all designated approvers to approve a request |
| Approver | An individual or entity responsible for reviewing and approving a request |
| Request | A document, process, or action that requires approval |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic.
The core concepts of the Approval Type Everyone Must Approve include:
* Unanimous consent: All approvers must approve the request for it to be considered approved
* Designated approvers: A predefined list of individuals or entities responsible for approving the request
* Approval workflow: A sequence of steps that defines the approval process

## 5. Standard Model
Describe the generally accepted or recommended model.
The standard model for the Approval Type Everyone Must Approve involves the following steps:
1. Request submission: The request is submitted for approval
2. Approval notification: Designated approvers are notified of the request
3. Approval review: Approvers review the request and provide their approval or rejection
4. Approval tracking: The approval status is tracked and updated in real-time
5. Final approval: The request is considered approved only when all approvers have provided their approval

## 6. Common Patterns
Document recurring, accepted patterns.
Common patterns associated with the Approval Type Everyone Must Approve include:
* Using this approval type for high-risk or high-impact decisions
* Implementing automated approval workflows to streamline the approval process
* Providing clear guidelines and criteria for approvers to follow

## 7. Anti-Patterns
Describe common but discouraged practices.
Anti-patterns associated with the Approval Type Everyone Must Approve include:
* Using this approval type for low-risk or routine decisions, which can lead to unnecessary delays and bottlenecks
* Failing to provide clear guidelines or criteria for approvers, leading to confusion and inconsistent approval decisions
* Not tracking or monitoring the approval status, resulting in lost or overlooked requests
