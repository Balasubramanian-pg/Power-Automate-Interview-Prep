# 108. Using Markdown in Approval Details

Canonical documentation for 108. Using Markdown in Approval Details. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 108. Using Markdown in Approval Details exists and the class of problems it addresses.
The primary purpose of using Markdown in approval details is to provide a standardized, readable, and easily parseable format for conveying complex information during the approval process. This addresses the problem of inefficient communication, which can lead to misunderstandings, delays, or even the rejection of requests due to a lack of clarity.

## 2. Conceptual Overview
Provide a high-level mental model of the topic.
Markdown is a lightweight markup language that allows users to create formatted text using plain text syntax. In the context of approval details, Markdown enables the inclusion of structured information such as headers, lists, links, and emphasis, making it easier for approvers to quickly understand and assess the request.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| Markdown | A lightweight markup language that allows users to create formatted text using plain text syntax. |
| Approval Details | The information provided to support a request for approval, which may include context, justification, and relevant data. |
| Formatter | A tool or software component that converts Markdown text into a visually formatted representation. |
| Parser | A software component that interprets Markdown syntax and generates a structured representation of the content. |
| Rendering | The process of displaying formatted text based on Markdown syntax. |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic.
The core concepts of using Markdown in approval details include:
- **Readability**: Markdown formatting should enhance the readability of approval details, making it easier for approvers to understand the request.
- **Consistency**: A consistent Markdown syntax should be used throughout the approval details to avoid confusion.
- ** Parsability**: The Markdown syntax should be easily parseable by tools and software, enabling automated processing and analysis of approval details.

## 5. Standard Model
Describe the generally accepted or recommended model.
The standard model for using Markdown in approval details involves:
1. **Header Sections**: Using headers (H1, H2, etc.) to organize and structure the approval details.
2. **Lists and Bullet Points**: Utilizing lists and bullet points to present multiple items or options.
3. **Emphasis and Strong Text**: Applying emphasis (italic) and strong text (bold) to draw attention to critical information.
4. **Links and References**: Including links to external resources or references to support the request.

## 6. Common Patterns
Document recurring, accepted patterns.
Common patterns for using Markdown in approval details include:
- **Request Summary**: Providing a brief summary of the request at the top, followed by detailed information.
- **Justification Section**: Including a dedicated section for justifying the request, using Markdown to format the text for clarity.
- **Data and Metrics**: Presenting data and metrics in a formatted table or list to facilitate easy understanding.

## 7. Anti-Patterns
Describe common but discouraged practices.
Anti-patterns to avoid when using Markdown in approval details include:
- **Overuse of Formatting**: Excessive use of Markdown syntax, which can make the text harder to read.
- **Inconsistent Syntax**: Mixing different Markdown syntax styles or not following a consistent formatting convention.
- **Lack of Clarity**: Failing to use Markdown to enhance the clarity and readability of the approval details.
