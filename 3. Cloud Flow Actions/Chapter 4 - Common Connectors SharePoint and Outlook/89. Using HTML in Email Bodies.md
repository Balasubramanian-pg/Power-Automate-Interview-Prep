# 89. Using HTML in Email Bodies

Canonical documentation for 89. Using HTML in Email Bodies. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 89. Using HTML in Email Bodies exists and the class of problems it addresses.
The purpose of using HTML in email bodies is to enhance the visual appeal and readability of emails, making them more engaging and effective in communicating their intended message. The problem space addressed by this topic includes the limitations of plain text emails, such as the inability to format text, add images, or create complex layouts. By using HTML in email bodies, senders can create more sophisticated and interactive emails that better capture the attention of recipients.

## 2. Conceptual Overview
Provide a high-level mental model of the topic.
The conceptual overview of using HTML in email bodies involves understanding the basic structure of an HTML document and how it can be applied to an email. This includes the use of HTML tags, such as `<html>`, `<head>`, and `<body>`, to define the content and layout of the email. Additionally, it involves considering the limitations and constraints of email clients, such as their ability to render HTML and CSS, and the need to ensure compatibility across different devices and platforms.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| HTML | HyperText Markup Language, a standard markup language used to create web pages and email content |
| Email Client | A software application used to send, receive, and manage email messages |
| CSS | Cascading Style Sheets, a styling language used to control the layout and appearance of HTML content |
| Inline CSS | CSS styles that are applied directly to HTML elements, rather than being defined in a separate stylesheet |
| Responsive Design | An approach to designing email content that adapts to different screen sizes and devices |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic.
The core concepts of using HTML in email bodies include understanding how to structure and format HTML content, how to use CSS to control the layout and appearance of the email, and how to ensure compatibility across different email clients and devices. Additionally, it involves considering the importance of accessibility, such as ensuring that the email can be read by screen readers and other assistive technologies.

## 5. Standard Model
Describe the generally accepted or recommended model.
The standard model for using HTML in email bodies involves using a simple and consistent structure, such as a single-column layout with a clear header and footer. It also involves using inline CSS to style the content, rather than relying on external stylesheets, and avoiding the use of complex layouts or interactive elements that may not be supported by all email clients.

## 6. Common Patterns
Document recurring, accepted patterns.
Common patterns for using HTML in email bodies include the use of tables to create complex layouts, the use of images to add visual interest, and the use of links to direct recipients to external websites or landing pages. Additionally, many email senders use templates or pre-designed layouts to create a consistent brand identity across their email campaigns.

## 7. Anti-Patterns
Describe common but discouraged practices.
Anti-patterns for using HTML in email bodies include the use of excessive or complex HTML code, the use of external stylesheets or scripts that may not be supported by all email clients, and the use of spammy or deceptive tactics, such as hiding text or using misleading subject lines.
