# 66. Iterating Over Arrays

Canonical documentation for 66. Iterating Over Arrays. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 66. Iterating Over Arrays exists and the class of problems it addresses.
Iterating over arrays is a fundamental concept in programming that enables developers to access and manipulate each element in a collection of data. The primary purpose of iterating over arrays is to perform operations on each element, such as data processing, transformation, or analysis. The class of problems it addresses includes, but is not limited to, data aggregation, filtering, sorting, and transformation.

## 2. Conceptual Overview
Provide a high-level mental model of the topic.
The conceptual model of iterating over arrays involves a loop construct that traverses the array, allowing developers to access and manipulate each element. This process can be visualized as a sequence of steps, where the loop iterates over the array, executing a set of instructions for each element. The loop can be controlled using various mechanisms, such as indices, pointers, or iterators, depending on the programming language and implementation.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| Array | A collection of elements of the same data type stored in contiguous memory locations. |
| Index | A numerical value that identifies the position of an element in an array. |
| Iterator | An object that enables traversal of a collection, such as an array, without exposing its underlying implementation. |
| Loop | A control structure that executes a set of instructions repeatedly for each element in an array. |
| Element | A single item in an array, which can be a primitive value or a complex object. |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic.
The core concepts of iterating over arrays include:
* **Loop constructs**: Mechanisms for controlling the iteration process, such as for loops, while loops, or foreach loops.
* **Array indexing**: The use of indices to access and manipulate elements in an array.
* **Iterator-based iteration**: The use of iterators to traverse an array without exposing its underlying implementation.
* **Element access**: The ability to access and manipulate individual elements in an array.

## 5. Standard Model
Describe the generally accepted or recommended model.
The standard model for iterating over arrays involves using a loop construct to traverse the array, accessing each element using its index or an iterator, and performing operations on the element as needed. This model is widely adopted across programming languages and is considered a best practice for iterating over arrays.

## 6. Common Patterns
Document recurring, accepted patterns.
Common patterns for iterating over arrays include:
* **Simple iteration**: Iterating over an array using a for loop or while loop to access each element.
* **Filtered iteration**: Iterating over an array and applying a filter condition to select specific elements.
* **Transformative iteration**: Iterating over an array and applying a transformation to each element.

## 7. Anti-Patterns
Describe common but discouraged practices.
Anti-patterns for iterating over arrays include:
* **Using unnecessary complexity**: Using overly complex loop constructs or iteration mechanisms when simpler alternatives are available.
* **Ignoring bounds checking**: Failing to check the bounds of an array before accessing its elements, leading to potential errors or exceptions.
* **Modifying the array during iteration**: Modifying the array while iterating over it, which can lead to unpredictable behavior or errors.
