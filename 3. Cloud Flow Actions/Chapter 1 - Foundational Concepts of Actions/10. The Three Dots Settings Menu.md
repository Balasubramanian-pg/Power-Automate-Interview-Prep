# 10. The Three Dots Settings Menu

Canonical documentation for 10. The Three Dots Settings Menu. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 10. The Three Dots Settings Menu exists and the class of problems it addresses.
The Three Dots Settings Menu exists to provide users with a convenient and accessible way to configure settings, access advanced features, and perform actions related to a specific context or application. The class of problems it addresses includes the need for a standardized and intuitive way to expose secondary actions, settings, and features that are not part of the primary user interface.

## 2. Conceptual Overview
Provide a high-level mental model of the topic.
The Three Dots Settings Menu is a contextual menu that is typically represented by an ellipsis (three dots) and is used to provide users with a list of available actions, settings, or features. The menu is usually hidden by default and is revealed when the user interacts with the three dots icon.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| Ellipsis | A symbol consisting of three dots (...) used to represent the menu |
| Context Menu | A menu that is specific to a particular context or application |
| Secondary Actions | Actions that are not part of the primary user interface but are still relevant to the user |
| Settings | Configuration options that allow users to customize the behavior of an application or feature |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic.
The core concepts of the Three Dots Settings Menu include:
* Contextual relevance: The menu should only contain actions, settings, or features that are relevant to the current context or application.
* Discoverability: The menu should be easily discoverable by users, with a clear and consistent visual representation.
* Consistency: The menu should follow a consistent design pattern and behavior across different applications and contexts.

## 5. Standard Model
Describe the generally accepted or recommended model.
The standard model for the Three Dots Settings Menu includes:
* The menu is represented by an ellipsis (three dots) icon.
* The menu is hidden by default and is revealed when the user interacts with the icon.
* The menu contains a list of available actions, settings, or features.
* The menu is contextual, meaning that it only contains items that are relevant to the current context or application.

## 6. Common Patterns
Document recurring, accepted patterns.
Common patterns for the Three Dots Settings Menu include:
* Using the menu to provide access to advanced features or settings.
* Using the menu to provide a way to perform actions that are not part of the primary user interface.
* Using the menu to provide a way to configure settings or options that are specific to a particular context or application.

## 7. Anti-Patterns
Describe common but discouraged practices.
Anti-patterns for the Three Dots Settings Menu include:
* Overloading the menu with too many items, making it difficult for users to find what they are looking for.
* Using the menu as a primary navigation mechanism, rather than as a secondary action.
* Failing to provide a clear and consistent visual representation of the menu.
