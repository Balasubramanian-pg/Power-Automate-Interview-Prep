# 110. User Friendly Error Messages in Apps

Canonical documentation for 110. User Friendly Error Messages in Apps. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 110. User Friendly Error Messages in Apps exists and the class of problems it addresses. The primary purpose of user-friendly error messages in apps is to provide clear, concise, and actionable feedback to users when an error occurs, helping them understand what went wrong and how to recover. This addresses the problem of user frustration, confusion, and abandonment that can result from cryptic or technical error messages.

## 2. Conceptual Overview
Provide a high-level mental model of the topic. User-friendly error messages in apps involve a combination of design principles, technical considerations, and user experience (UX) best practices. The goal is to create error messages that are easy to understand, non-technical, and empowering, allowing users to quickly resolve issues and continue using the app.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| Error Message | A notification or alert displayed to the user when an error occurs, providing information about the issue and potential solutions. |
| User-Friendly | Design principles and practices that prioritize clarity, simplicity, and usability in error messages, making them accessible to a broad audience. |
| Error Handling | The process of detecting, reporting, and resolving errors in an app, including the display of error messages to the user. |
| Technical Debt | The accumulation of design or implementation flaws in an app that can lead to errors and negatively impact the user experience. |
| Accessibility | The design and development of error messages that can be easily understood and used by people with disabilities, language barriers, or varying levels of technical expertise. |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic. The core concepts of user-friendly error messages in apps include:
* Clear and concise language
* Non-technical terminology
* Empathetic tone
* Actionable feedback
* Consistent design and formatting
* Accessibility and inclusivity

## 5. Standard Model
Describe the generally accepted or recommended model. The standard model for user-friendly error messages in apps involves:
1. **Error detection**: Identifying and logging errors in the app.
2. **Error reporting**: Displaying a clear and concise error message to the user.
3. **Error resolution**: Providing actionable feedback and potential solutions to the user.
4. **Error prevention**: Implementing design and development best practices to minimize errors and improve the overall user experience.

## 6. Common Patterns
Document recurring, accepted patterns. Common patterns for user-friendly error messages in apps include:
* Using a clear and concise header or title
* Providing a brief description of the error
* Offering actionable feedback or potential solutions
* Including a call-to-action (CTA) or next steps
* Using visual elements, such as icons or images, to enhance clarity and engagement

## 7. Anti-Patterns
Describe common but discouraged practices. Anti-patterns for user-friendly error messages in apps include:
* Using technical jargon or complex terminology
* Displaying generic or unhelpful error messages
* Failing to provide actionable feedback or potential solutions
* Using a negative or confrontational tone
* Ignoring accessibility and inclusivity considerations
