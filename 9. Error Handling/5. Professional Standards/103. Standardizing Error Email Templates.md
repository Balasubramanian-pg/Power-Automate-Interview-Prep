# 103. Standardizing Error Email Templates

Canonical documentation for 103. Standardizing Error Email Templates. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 103. Standardizing Error Email Templates exists and the class of problems it addresses.
The purpose of standardizing error email templates is to provide a consistent and efficient way to communicate errors to users, reducing confusion and improving the overall user experience. The problem space includes the lack of consistency in error messaging, inadequate information provided to users, and the need for a scalable and maintainable solution.

## 2. Conceptual Overview
Provide a high-level mental model of the topic.
Standardizing error email templates involves creating a set of predefined templates that can be used to communicate different types of errors to users. These templates should include essential information such as error description, steps to resolve the issue, and contact information for further assistance. The templates should be flexible enough to accommodate different error scenarios and customizable to fit the organization's brand and tone.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| Error Email Template | A predefined template used to communicate errors to users via email. |
| Error Type | A category of errors, such as authentication errors or payment errors. |
| Template Variable | A placeholder in the template that can be replaced with dynamic content, such as the user's name or error code. |
| Error Code | A unique identifier for a specific error, used for tracking and debugging purposes. |
| Error Message | A human-readable description of the error, providing context and instructions for resolution. |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic.
The core concepts of standardizing error email templates include:
* Consistency: Using a standardized template ensures that error messages are consistent in format and tone, making it easier for users to understand and respond to errors.
* Flexibility: Templates should be flexible enough to accommodate different error scenarios and customizable to fit the organization's brand and tone.
* Scalability: Standardized templates enable efficient management and maintenance of error messaging, reducing the burden on support teams and improving response times.
* User Experience: Error email templates should prioritize the user experience, providing clear and concise information to help users resolve issues quickly and efficiently.

## 5. Standard Model
Describe the generally accepted or recommended model.
The standard model for error email templates includes the following elements:
* Introduction: A brief introduction explaining the purpose of the email and the error that occurred.
* Error Description: A clear and concise description of the error, including any relevant details or codes.
* Steps to Resolve: Instructions on how to resolve the issue, including any necessary actions or troubleshooting steps.
* Contact Information: Contact information for further assistance, including email addresses, phone numbers, or support portals.
* Closing: A polite closing and any additional information or resources that may be helpful to the user.

## 6. Common Patterns
Document recurring, accepted patterns.
Common patterns for error email templates include:
* Using a clear and concise subject line that indicates the error type and severity.
* Including a clear call-to-action (CTA) that instructs the user on the next steps to take.
* Providing additional resources or troubleshooting steps to help the user resolve the issue.
* Using a standardized format for error codes and messages to facilitate tracking and debugging.

## 7. Anti-Patterns
Describe common but discouraged practices.
Anti-patterns for error email templates include:
* Using generic or vague error messages that do not provide sufficient information to the user.
* Including too much technical jargon or complex terminology that may confuse the user.
* Failing to provide clear instructions or next steps for the user to take.
* Using an unprofessional or unfriendly tone that may alienate or frustrate the user.
